# Carpet-Pvp-Bots

## korean
마인크래프트 페브릭 모드 중 하나인 Carpet을 이용하여 만들어진 간단한 PVP 봇 입니다.
현재까지 만들어진 기능은 다음과 같습니다.

| function | percent | tested | working perfectly |
| -------- | ------ | ------- | ----------------- |
| Attack | 100% | Y | Y |
| Attack with looking player | 100% | Y | Y |
| Following player | 90% | Y | 90% |
| Stacks block if player is higher than bot | 80% | Y | 80% |
| Opens door | 90% | Y | 90% |
---

플레이어를 따라가 공격하는 것과 참나무 문이 앞에 있다면 여는 것, 플레이어가 본인보다 위에 있을 경우 블록을 쌓아 올라가는 것이 구현되어 있습니다.
댜만, 따라가는 과정이 그저 플레이어룰 바라보며 가는것으로 끝나므로 복잡한 곳이나 미로 등에서 정상적으로 따라가지 못 합니다.
이를 방지해 "Anti-Stuck" (끼임 방지) 기능이 구현되어 있으나 해당 기능은 그저 봇이 특정 구역에 끼어있다고 판단되는 경우, 봇을 정지 처리하여 봇이 이상한 곳으로 가지 못 하게 하는것에 그칩니다.

이후에 A* 알고리즘을 시도해 볼 에정이나, 언제가 될지 모릅니다.

## 필요 항목
- JE (1.21.1)
- Carpet, Latest

마인크래프트의 버전에 경우 1.21.11 이하여도 충분히 구동이 가능하나 (설게상) 맵 로딩 과정에서 문제가 발생할 수 있으므로 되도록이면 1.21.11 이여야 합니다.

## 준비

**경고**

해당 맵의 데이터팩, 커맨드 블록 속 명령어는 저의 플레이어 ID만을 가르키고 있습니다. **실 사용을 위해서는 먼저 커맨드 블록 속 명령어를 비롯하여 월드 내부 데이터팩 또한 수동으로 수정해야 합니다.**

---

1. `/player Alex spawn` 명령어를 사용하여 알렉스를 생성 합니다. (생성 전에 에러 메시지가 지속적으로 나올 수 있으나, 정상적인 반응 입니다.
2. 빨간색 양털이 있는 곳을 레드스톤 블록으로 바꾸면 즉시 시작됩니다.

---

봇이 블록을 쌓는 과정에서 내부적으로 오류가 발생하면 그대로 멈출 수 있습니다. 그런경우, `/function status:reset` 명령어를 통해 재시작하면 해결 됩니다.
